## Things to NOT do

- Do **not** prepend `/bin/sh -c` or any shell — `cmd[0]` is
  invoked directly via `os/exec` (FR-020-1, R-001).
- Do **not** call `LookPath` — relative-path resolution is
  forbidden (FR-020-3, R-002).
- Do **not** add a `Restart()` method — `Child` is single-use
  (FR-020-11).
- Do **not** add an `ErrChildExited` sentinel — the single
  `ErrChildNotStarted` covers every "no live child" case
  (Clarification 2, R-011).
- Do **not** make `Wait` ctx-cancellable — `cmd.Wait()` is
  uncancellable; cancellation flows through `Forward(SIGTERM)`
  (R-013).
- Do **not** allow `cfg.Logger == nil` — panic at `NewChild`
  (Constitution IX startup-wiring exemption, R-002).
- Do **not** spawn the forwarding/drain/death-watch goroutines
  outside `Start` — every per-`Start` goroutine is joined via
  `c.wg` released in `Wait` (FR-020-7, R-013, R-014).
- Do **not** add a fuzz target (R-008).
- Do **not** introduce a separate `cmd/test-helper-supervise`
  binary — use `os.Executable()` re-invocation (R-012, R-015).
- Do **not** introduce `internal/testutil` helpers — fakes
  inline in test files (R-015, mirrors SDD-18/SDD-19 R-011).
- Do **not** alter any SDD-18 (`config/`) or SDD-19 (`state.go`)
  symbol — both are out of scope for this chunk.
- Do **not** initiate any I/O outside the child boundary —
  no network, no vault access, no Discord, no audit-log
  writes, no SDD-19 state-machine `Transition` calls
  (FR-020-16). The wiring from `Wait`'s exit triple to
  SDD-19's `EventChildExit*` events lives in SDD-21.
- Do **not** claim Darwin's death-watch handles
  SIGKILL-of-supervisor — it does not. The R-009 gap is
  documented in research.md and surfaced via a
  `t.Skip("R-009 darwin gap")` subtest in T-11b.

## Known gaps documented for v0.1.1

- **R-009 — Darwin SIGKILL gap**: A SIGKILL'd supervisor cannot
  trigger the in-process kqueue goroutine. Linux Pdeathsig
  covers this case via the kernel; Darwin does not. v0.1.1 may
  add a guardian-subprocess pattern for full coverage on
  darwin. The lifecycle harness (SDD-25) skips the relevant
  test case on darwin.

This is the only documented limitation in this chunk. Every
other contract is honoured fully on both platforms.
