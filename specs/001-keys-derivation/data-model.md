# Phase 1 Data Model — internal/keys (SDD-01)

`internal/keys` is a stateless leaf package; it has no persisted
entities. The "data model" here is the typed surface that flows
across the package's exported boundary. Every entry below pins:

- the Go type used,
- whether the value is **secret** (must never appear in logs or as a
  Go `string` per Constitution X),
- the lifetime / ownership rule.

No struct types are defined or persisted by this package. Callers
own the memory.

---

## Inputs

### Passphrase

| Field | Value |
|-------|-------|
| Go type | `[]byte` (parameter to `DeriveMasterSeed`) |
| Constraints | `len ≥ 12`. Bytes opaque (no character-set restriction). |
| Validation rule | `len(passphrase) < 12` ⇒ `ErrPassphraseTooShort`. |
| Secret? | **Yes.** Never logged, never converted to `string` inside this package. |
| Lifetime | Caller-allocated and caller-zeroed. This package never retains a reference past return. |
| Source | macOS Keychain on the trusted host (`hush` keychain entry, ACL-restricted). Acquired upstream — outside this package's scope. |

### Salt

| Field | Value |
|-------|-------|
| Go type | `[]byte` (parameter to `DeriveMasterSeed`) |
| Constraints | `len == 16` exactly. Any other length (0, 8, 15, 17, 24, 32, …) is rejected. |
| Validation rule | `len(salt) != 16` ⇒ `ErrSaltMissing`. |
| Secret? | **No.** Stored alongside the vault file in plaintext (per the vault format defined in SDD-03). Treated as input parameter only. |
| Lifetime | Caller-allocated; not retained after return. |
| Source | The vault file's salt field (16 bytes, generated by `crypto/rand` in SDD-03 / SDD-15 init flow). Outside this package's scope. |

### Machine index

| Field | Value |
|-------|-------|
| Go type | `uint32` (parameter to `DeriveClientKey`) |
| Constraints | Any value in `[0, 2^32)` is valid. No range restriction beyond the type. |
| Validation rule | None — every `uint32` value yields a valid keypair. The derivation MUST NOT silently truncate or wrap (Edge case: "Maximum machine index"). |
| Secret? | **No.** Public identifier shared with the vault server's allowed-clients config. |
| Lifetime | Pass-by-value; trivially scoped to the call. |

### Public key (for fingerprint)

| Field | Value |
|-------|-------|
| Go type | `*ecdsa.PublicKey` (parameter to `PublicKeyFingerprint`) |
| Constraints | Curve must be secp256k1 (the only curve produced by this package's derivations). Public points outside the curve are not handled — caller responsibility. |
| Validation rule | None enforced by this package. Calling with a non-secp256k1 key is undefined behaviour outside the package contract. |
| Secret? | **No.** Public material by construction. |
| Lifetime | Read-only; not retained. |

---

## Outputs

### Master seed

| Field | Value |
|-------|-------|
| Go type | `[]byte` (return of `DeriveMasterSeed`) |
| Length | Exactly 64 bytes (`Argon2id keyLen=64`). |
| Determinism | Same `(passphrase, salt)` ⇒ byte-identical seed across processes / OSes / CPU architectures. |
| Secret? | **Yes.** The root of the entire hush key tree. Never logged; never converted to `string`. |
| Lifetime | Returned to caller. This package retains no copy. The caller is expected to wrap it in `internal/vault.SecureBytes` (SDD-02) for mlock + zero-on-free. |

### JWT signing key

| Field | Value |
|-------|-------|
| Go type | `*ecdsa.PrivateKey` (return of `DeriveJWTSigningKey`) |
| Curve | secp256k1 (Bitcoin curve). |
| BIP32 path | `m/44'/7743'/0'`. |
| Determinism | Same seed ⇒ identical key across processes / OSes / CPU architectures. |
| Secret? | **Yes** (private scalar `D`). The `PublicKey` field is non-secret and shareable. |
| Lifetime | Returned to caller; consumer is `internal/token` (SDD-07). |
| Distinctness | MUST differ from the audit signing key (FR-007 explicit). |

### Vault encryption key

| Field | Value |
|-------|-------|
| Go type | `[]byte` (return of `DeriveVaultEncKey`) |
| Length | Exactly 32 bytes (AES-256 key). |
| Source | The 32-byte private scalar of the BIP32 child node at `m/44'/7743'/1'`, taken directly. |
| Determinism | Same seed ⇒ identical key across processes / OSes / CPU architectures. |
| Secret? | **Yes.** Used as the AES-256-GCM key in SDD-03 vault format. |
| Lifetime | Returned to caller; consumer is `internal/vault` (SDD-03). |

### Audit signing key

| Field | Value |
|-------|-------|
| Go type | `*ecdsa.PrivateKey` (return of `DeriveAuditSigningKey`) |
| Curve | secp256k1. |
| BIP32 path | `m/44'/7743'/2'`. |
| Determinism | Same seed ⇒ identical key. |
| Secret? | **Yes** (private scalar). |
| Lifetime | Returned to caller; consumer is `internal/logging` audit-writer (SDD-05). |
| Distinctness | MUST differ from the JWT signing key (FR-007 + AC-7 row in `docs/AC-MATRIX.md`). |

### Per-machine client keypair

| Field | Value |
|-------|-------|
| Go type | `*ecdsa.PrivateKey` (return of `DeriveClientKey(seed, machineIndex)`) |
| Curve | secp256k1. |
| BIP32 path | `m/44'/7743'/3'/{machineIndex}` (hardened only at `3'`; the leaf is non-hardened so machine index ranges over the full `uint32` space without overflow). |
| Determinism | Same `(seed, machineIndex)` ⇒ identical key. |
| Distinctness | Distinct `machineIndex` values ⇒ distinct keypairs (User Story 2). Distinct `seed` values (different passphrases) ⇒ distinct keypair even at the same `machineIndex` (AC-6). |
| Secret? | **Yes** (private scalar). The public half is shared with the server in `registered_client_keys`. |
| Lifetime | Returned to caller; consumer is `internal/transport/sign` (SDD-08) and `hush init --client` (SDD-15 / SDD-16). |

### Public-key fingerprint

| Field | Value |
|-------|-------|
| Go type | `string` (return of `PublicKeyFingerprint`) |
| Length | Exactly 16 characters, lowercase hex. |
| Algorithm | `hex.EncodeToString(sha256(SEC1_compressed(pub))[:8])`. |
| Determinism | Same public key ⇒ identical fingerprint. |
| Secret? | **No.** Operates on a public key only. The `string` return type is permitted because the input and output are both non-secret. (The Constitution X anti-rule on `[]byte → string` applies only to **secret** material.) |
| Lifetime | Returned by value; not retained. |

## Errors

### `ErrPassphraseTooShort`

- **Type:** `error` (sentinel; package-level `var = errors.New("hush/keys: passphrase too short")`).
- **Returned by:** `DeriveMasterSeed`.
- **Trigger:** `len(passphrase) < 12`.
- **Latency:** `< 100ms` (must precede Argon2id call) — SC-004.
- **Leakage:** error message identifies the failure mode only. It does NOT echo the passphrase length (avoids fingerprinting weak inputs).

### `ErrSaltMissing`

- **Type:** `error` (sentinel; package-level `var = errors.New("hush/keys: salt missing or wrong length")`).
- **Returned by:** `DeriveMasterSeed`.
- **Trigger:** `len(salt) != 16`.
- **Latency:** `< 100ms`.
- **Leakage:** identifies the failure mode only.

### `ctx.Err()` (passthrough)

- **Type:** whatever the caller's context returned (`context.Canceled`, `context.DeadlineExceeded`, etc.). Compared with `errors.Is`.
- **Returned by:** `DeriveMasterSeed`.
- **Trigger:** `ctx.Err() != nil` at function entry. Never re-checked after Argon2id starts (FR-013).
- **Leakage:** none.

---

## State machine

There is no state machine. The package is stateless and every
exported function is pure (output depends solely on inputs).

The lifecycle is therefore trivial:

```
caller → DeriveMasterSeed(ctx, passphrase, salt) → seed
caller → DeriveJWTSigningKey(seed)                → *ecdsa.PrivateKey
caller → DeriveVaultEncKey(seed)                  → []byte (32B)
caller → DeriveAuditSigningKey(seed)              → *ecdsa.PrivateKey
caller → DeriveClientKey(seed, machineIndex)      → *ecdsa.PrivateKey
caller → PublicKeyFingerprint(pub)                → string (16 hex)
```

Each call is independent. The package retains nothing between calls.
