# Implementation Plan: CLI Root + Server-Facing Subcommands

**Branch**: `014-cli-root-and-server-cmds` | **Date**: 2026-05-01 | **Spec**: [spec.md](./spec.md)
**Input**: [docs/sdd/SDD-14.md](../../docs/sdd/SDD-14.md), [docs/SPEC.md](../../docs/SPEC.md), [docs/API.md](../../docs/API.md), [docs/PACKAGE-MAP.md](../../docs/PACKAGE-MAP.md), [.specify/memory/constitution.md](../../.specify/memory/constitution.md)

## Summary

Land the `hush` binary's public surface: `cmd/hush/main.go` plus the `internal/cli` root command and the four operator-facing subcommands (`serve`, `health`, `version`, `revoke`). Deliverables are (1) a cobra root that wires the four global flags `--config`, `--verbose`, `--quiet`, `--no-color` identically to every subcommand, (2) a per-stream output formatter that picks human text on a TTY and stable JSON on a pipe — never injecting ANSI when `--no-color` is set, (3) a fixed exit-code contract (`ExitOK=0, ExitErr=1, ExitInputErr=2, ExitAuth=3, ExitNotFound=4, ExitPerm=5, ExitConfigStale=78`) emitted via `cli.Execute() int` so `cmd/hush/main.go` can `os.Exit` with no other logic, and (4) the four subcommands themselves. `serve` resolves the vault passphrase strictly via `stdin pipe → TTY prompt → ExitInputErr` (POSIX-line semantics: strip exactly one trailing `\n` or `\r\n`, preserve all other bytes), then composes the chassis from already-locked surfaces (`internal/keys` → `internal/vault` → `internal/token` → `internal/audit` → `internal/server`) and propagates `(*Server).Run`'s terminal error to the right exit code; SIGTERM/SIGINT trigger graceful shutdown via the chassis's own context cancellation. `health` issues a 5-second-bounded `GET /hz`, prints the per-dimension summary in the right shape for the output context, and exits `ExitOK` only when every dimension is healthy. `revoke` constructs `{jti, nonce, timestamp}`, signs it through `internal/transport/sign.CanonicalJSON`+`Sign`, posts to `/revoke` with the same 5-second timeout, and maps HTTP status to exit code. `version` prints either human metadata or the locked `{"version","commit","date"}` JSON shape with `dev`/`unknown` placeholders for unbuilt fields. Constitution VII (cobra-only — no viper), IX (idiomatic Go, ctx-first, no globals/init for runtime state), X (zero secret bytes anywhere — verified by sentinel-leak tests using `SECRET_SHOULD_NEVER_APPEAR_14`), and XI (one new direct dep `github.com/spf13/cobra`; one new test-only dep `github.com/creack/pty`; both justified in `research.md`) are load-bearing. Coverage target ≥ 85% on `internal/cli`; the passphrase-resolution and sentinel-leak paths reach 100%.

## Technical Context

**Language/Version**: Go 1.26.1 (per `go.mod`).
**Primary Dependencies**: `github.com/spf13/cobra` v1.8+ (NEW direct dep — Constitution VII mandates cobra; XI justification in `research.md`); `golang.org/x/term` v0.42.0 (already on tree — used for `IsTerminal` + `ReadPassword`); `internal/cli` consumes already-locked APIs from `internal/config` (`LoadServer`, the seven sentinels), `internal/keys` (`DeriveMasterSeed`, `DeriveJWTSigningKey`, `DeriveVaultEncKey`, `DeriveAuditSigningKey`), `internal/vault` (`Load`, `Store.Names`, `ErrSecretNotFound`), `internal/vault/securebytes` (`New`, `Use`, `Destroy`), `internal/token` (`NewStore`, `Issue`/`Validate` signatures only via type — issuance happens server-side), `internal/transport/sign` (`CanonicalJSON`, `Sign`), `internal/transport/ecies` (NOT consumed by this chunk — `serve` only constructs the chassis), `internal/audit` (`NewWriter`, `Run`, `DiscordMirror`), `internal/discord` (`NewBotApprover`, `BotConfig`, `Connected()`), `internal/server` (`New`, `Deps`, `Run`, `RegisterHandlers`, `ReloadVault`, the locked sentinels), `internal/logging` (`New`, `Format`). NEW direct dep `github.com/spf13/cobra`; NEW test-only dep `github.com/creack/pty` (research-justified — sole consumer is the no-echo passphrase test; no production import).
**Storage**: None added. `serve` opens the vault file at `cfg.Server.StateDir + "/secrets.vault"` via `internal/vault.Load`. `health` and `revoke` read no local files except the config file passed via `--config`. `version` reads no files. The audit log file is opened by `internal/audit.NewWriter` inside `serve`'s wiring path.
**Testing**: `go test -race ./internal/cli/...` for unit; `//go:build integration` for `TestServe_StartAndShutdown` (which spins the chassis with a `testutil.DiscordStub`-class approver and an in-memory audit mirror, sends `SIGTERM`, and asserts clean exit). The pseudo-tty test (`TestServe_PassphraseFromTTYPrompt`) uses `creack/pty` to drive `term.ReadPassword` because no other primitive proves the no-echo property faithfully. `TestServe_NeverReadsEnv` is a static check that scans every `internal/cli/*.go` source file for the literal token `os.Getenv` and fails on any hit (FR-009 verifiability).
**Target Platform**: macOS 12+ (Darwin) and Linux on the operator's vault host. The `creack/pty` dependency is unix-only, which is acceptable: `serve`'s no-echo test runs on the same operator-host platforms the binary targets.
**Project Type**: Single Go module (`github.com/mrz1836/hush`). New packages: `cmd/hush` (the only `main` in the repo) is filled, and `internal/cli` gains the locked file set listed under "Source Code".
**Performance Goals**: AC-1 + SC-001 — `hush serve` must accept `/hz` requests within 5 s of invocation excluding passphrase delivery; the CLI's contribution to that budget is ≤ 200 ms (config load + key derivation excluded — those run on the chassis path, governed by Argon2id NFR-5). `health` and `revoke` complete their one HTTP call within their own 5 s timeout (FR-015a clarification).
**Constraints**: Constitution VII (cobra subcommands; the four global flags; the seven exit codes; TTY-aware output); IX (ctx-first; no `init()`; no globals carrying mutable runtime state; goroutine ownership obvious — `serve` owns one signal-listener goroutine via `signal.NotifyContext`); X (no secret value, no JWT byte, no signing-key byte, no ECIES envelope byte ever appears in any output stream — verified by `TestServe_OutputNoSentinel`, `TestRevoke_OutputNoSentinel`, `TestHealth_OutputNoSentinel`); XI (one new direct dep + one test-only dep, both justified; `govulncheck` runs in CI on the new tree). Coverage 85% overall on `internal/cli`; passphrase-resolution and sentinel-leak paths must reach 100%.
**Scale/Scope**: Two new packages (`cmd/hush` + `internal/cli`), 14 source/test files locked by SDD-14, ~1.2k LOC including tests. No business logic concentration outside `internal/cli`; `cmd/hush/main.go` is two lines of body.

| **IX. Idiomatic Go discipline** | ✅ Pass — load-bearing | Every public function takes `context.Context` first when it does I/O — `cli.Execute(ctx context.Context) int`, `serve.run(ctx)`, `health.run(ctx)`, `revoke.run(ctx)` — and never stores ctx in a struct field. The integration test seam: `internal/cli/serve.go` exposes one **unexported** function `runServe(ctx, deps serveDeps) error` where `serveDeps` is an unexported struct with the testable surfaces (passphrase source, approver constructor, audit-writer constructor, listener factory). The default `serveCmd.RunE` builds production `serveDeps` from the loaded config; the integration test builds test `serveDeps` with `testutil.DiscordStub`, an in-memory `audit.MirrorSession`, and a free-port listener. This pattern matches the chassis's own `Deps`-injection style and does not export new symbols beyond `Execute` and the seven exit codes (per the locked exported-API contract).
