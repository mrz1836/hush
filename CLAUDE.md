Active feature plan: [specs/015-init-and-keychain/plan.md](specs/015-init-and-keychain/plan.md) — SDD-15 `hush init` server + client modes + new `internal/keychain` package. Two cobra subcommands (`hush init server`, `hush init client --machine-index N`) under the SDD-14 root; mutual exclusivity enforced **structurally** by the command tree (no flag-combo conflicts possible). Strictly TTY-only: passphrase, confirmation (double entry, both modes), and Discord bot token all read via `golang.org/x/term.ReadPassword`; non-TTY stdin → `ExitInputErr` with locked text "stdin must be an interactive terminal". **Never** `os.Getenv` (lint check), never reads any flag value as a passphrase, never generates a passphrase. Passphrase ≥ 12 bytes (rejected before any KDF); 16-byte salt from `crypto/rand`. `server`: existence-guards (vault, config, both keychain items) → derive master seed (`keys.DeriveMasterSeed`) → `vault.Save` empty store at `<state_dir>/secrets.vault` (mode 0600) → atomic-write `<state_dir>/config.toml` at mode 0600 with **every** `docs/CONFIG-SCHEMA.md` default plus operator-prompted `listen_addr`/`discord_owner_id`/`application_id` (path_prefix generated 12-char `crypto/rand` URL-safe) → `Keychain.Store` for vault passphrase (`hush-vault-passphrase`/`hush-server`) and bot token (`cfg.Discord.BotTokenKeychainItem`/`hush-server`); round-trip-validate via `config.LoadServer`. `client`: requires `--machine-index N` (uint32) → derive client key (`keys.DeriveClientKey`) → `Keychain.Store` at (`hush-client`, `machine-<N>`) → print exactly one `SHA256:<43-char-base64-no-padding>` line to stdout (50 chars + `\n`). Both modes pass `os.Executable()` absolute path as the `acl` argument. `internal/keychain` (NEW): `Keychain` interface (`Store`/`Retrieve`/`Delete`), `New(*slog.Logger)` factory (darwin: shells `/usr/bin/security` with `-T <abs-path>` and `-w` stdin to avoid argv exposure; linux: `github.com/zalando/go-keyring` wrapper for build only), `PerBinaryACLSupported() bool` capability probe (true on darwin, false on linux), sentinels `ErrKeychainItemNotFound`/`ErrKeychainItemExists`/`ErrKeychainPermissionDenied`/`ErrKeychainUnsupportedPlatform`, test-only `FakeKeychain` with recorded ACL for assertions. **FR-020a**: init refuses on Linux today (Secret Service has no per-binary ACL) with locked message "platform linux has no per-binary keychain ACL; init refuses to run" → `ExitErr`; no silent downgrade, no opt-in flag. **Re-run discipline (Clarification Q1)**: pre-existing vault/config/keychain item → refuse-and-exit; operator must delete by hand. **Fingerprint**: new local helper in `init.go` computes `SHA256:base64.RawStdEncoding(sha256(SEC1Compressed(pub)))` — keys.PublicKeyFingerprint (16-char hex, SDD-01-locked) is **NOT** modified. Atomic-write pattern: `O_WRONLY|O_CREATE|O_EXCL 0o600` → `f.Sync()` → `os.Rename` → defensive `os.Chmod(0o600)`. **NEW direct dep** (Constitution XI written justification, research §1): `github.com/zalando/go-keyring` MIT, transitive `godbus/godbus/v5` only, no cgo, alternatives (hand-roll D-Bus / shell `secret-tool` / `keybase/go-keychain` (cgo) / `99designs/keyring` (10 transitive deps)) rejected.
